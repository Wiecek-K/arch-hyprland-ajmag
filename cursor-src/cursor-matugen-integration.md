# Cursor — matugen integration plan

Current state: Vimix cursors (dark variant) compiled to hyprcursor format, installed at
`~/.local/share/icons/Vimix-hyprcursor/`. XCursor fallback (XWayland/Steam) installed at
`~/.local/share/icons/Vimix-cursors/`. SVG source at `~/cursor-src/Vimix-cursors/`.

## Installed sizes

| Layer | Theme | Size | Config location |
|-------|-------|------|----------------|
| Hyprland (hyprcursor) | `Vimix-hyprcursor` | 36 | `hyprland.conf` env |
| XWayland/Steam (xcursor) | `Vimix-cursors` | 36 | `hyprland.conf` env |
| GTK apps (Waybar, Nautilus…) | `Vimix-cursors` | 24 | `gsettings` + `gtk-3.0/settings.ini` + `gtk-4.0/settings.ini` |

GTK size differs from hyprcursor — 36 na GTK wyglądało za duże na Waybarze (pasek 42px).

Zmiana rozmiaru GTK wymaga trzech miejsc jednocześnie:
```bash
gsettings set org.gnome.desktop.interface cursor-size <SIZE>
sed -i "s/gtk-cursor-theme-size=.*/gtk-cursor-theme-size=<SIZE>/" ~/.config/gtk-3.0/settings.ini ~/.config/gtk-4.0/settings.ini
~/.config/hypr/scripts/wbrestart.sh
```

Zmiana rozmiaru hyprcursor (desktop):
```bash
hyprctl setcursor Vimix-hyprcursor <SIZE>
# + zaktualizuj XCURSOR_SIZE i HYPRCURSOR_SIZE w hyprland.conf
```

Cursor hide timeout: `cursor:inactive_timeout = 10` w `hyprland.conf`.

## SVG source colors (Vimix dark)

| Role | Hex | Where used |
|------|-----|-----------|
| Cursor body | `#333333` | `fill` on main shape paths |
| Inner highlight | `#ffffff` | `fill` on highlight/arrow interior |
| Deep shadow | `#1a1a1a` | shadow/outline paths |
| Drop shadow | `#000000` | blur filter paths (opacity ~0.35) |

Only `#333333` and `#ffffff` are worth targeting — the shadow paths are near-black regardless of palette.

## What needs to change for matugen integration

### 1. SVG source repo

Keep unmodified Vimix SVG source at `~/cursor-src/Vimix-cursors/src/svg/`.
Never edit files there manually — they are the reset baseline.

### 2. matugen template

Create `~/.config/matugen/templates/cursor-colors.sh` that exports two variables:

```bash
# generated by matugen — do not edit
CURSOR_BODY="{{colors.dark.on_surface_variant}}"   # replaces #333333
CURSOR_HIGHLIGHT="{{colors.dark.surface}}"          # replaces #ffffff
```

Add entry to `~/.config/matugen/config.toml`:

```toml
[templates.cursor_colors]
input_path  = "~/.config/matugen/templates/cursor-colors.sh"
output_path = "~/.config/matugen/generated/cursor-colors.sh"
```

### 3. Rebuild script

Create `~/.config/hypr/scripts/cursor-rebuild.sh`:

```bash
#!/usr/bin/env bash
set -e

SRC=~/cursor-src/Vimix-cursors/src/svg
WORK=/tmp/cursor-build/src/svg
COMPILED=/tmp/cursor-build-out
DEST=~/.local/share/icons/Vimix-hyprcursor

source ~/.config/matugen/generated/cursor-colors.sh

# Fresh working copy
rm -rf /tmp/cursor-build
cp -r ~/cursor-src/Vimix-cursors /tmp/cursor-build

# Substitute colors
find "$WORK" -name "*.svg" | xargs sed -i \
  -e "s/#333333/$CURSOR_BODY/gI" \
  -e "s/#ffffff/$CURSOR_HIGHLIGHT/gI"

# Compile (requires compiled working state from prior hyprcursor-util --extract run)
# If working with hyprcursor source directly:
hyprcursor-util --create /tmp/cursor-build-working -o "$COMPILED"
cp -r "$COMPILED"/create_Vimix "$DEST"

# Activate without restarting Hyprland
hyprctl setcursor Vimix-hyprcursor 24
```

### 4. Hook into wppicker.sh

`~/.config/hypr/scripts/wppicker.sh` already calls `matugen image` after wallpaper selection.
Add one line after the matugen call:

```bash
~/.config/hypr/scripts/cursor-rebuild.sh &
```

The `&` keeps wallpaper switch instant; cursor updates in background (~1-2s).

### 5. hyprland.conf env vars (already set after migration)

```ini
env = XCURSOR_SIZE,36
env = XCURSOR_THEME,Vimix-cursors
env = HYPRCURSOR_SIZE,36
env = HYPRCURSOR_THEME,Vimix-hyprcursor
```

## Matugen color token guidance

Vimix dark uses neutral dark tones. Good matugen token candidates:

| SVG role | Recommended token | Notes |
|----------|------------------|-------|
| Body (`#333333`) | `colors.dark.on_surface_variant` | neutral, readable on any bg |
| Highlight (`#ffffff`) | `colors.dark.surface` | light surface — keeps contrast |

Alternative: use `colors.dark.primary` for body to get accent-colored cursor.
Avoid `colors.dark.on_primary` — too saturated, reduces readability.

## Prerequisites before running the integration

- `hyprcursor-util` installed (`pacman -S hyprcursor`) ✓
- SVG source w `~/cursor-src/Vimix-cursors/` ✓ (git clone, nie AUR)
- Build script: `/tmp/.../build_hyprcursor.py` — przy rebuildzie skopiować do `~/cursor-src/`
- `jq` available for parsing matugen JSON (optional if using template approach above)
